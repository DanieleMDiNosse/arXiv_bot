# Scholar Stream

`Scholar Stream` is a Telegram bot for researchers who want a lightweight way to monitor fresh papers across multiple academic sources without building a full alerting pipeline.

It tracks new results from **arXiv**, **bioRxiv**, **medRxiv**, **ChemRxiv**, **SSRN**, **IEEE**, and **PubMed**, using keyword lists defined by each user. Results are delivered in Telegram with compact metadata, expandable abstracts, direct paper links, PDF download buttons when available, bookmarks, and scheduled recaps.

## Supported sources

- arXiv
- bioRxiv
- medRxiv
- ChemRxiv
- SSRN
- IEEE
- PubMed

## What the bot does

- Search the **last 24 hours** by default, or any custom number of hours
- Keep **separate keyword lists per source**
- Support simple Boolean-style keyword input: commas = OR, `+` = AND inside one clause
- Send **daily recaps** at one or more local times with a selectable timezone
- Save papers to **bookmarks** for later review
- Let users **report issues** directly from Telegram

Example keyword input:

```text
"quantum mechanics" + entanglement, superconductivity
```

This means: find papers matching `quantum mechanics AND entanglement`, or matching `superconductivity`.

## User experience

The bot is designed to work from Telegram's reply keyboard, so non-technical users do not need to remember slash commands. The main workflow is:

1. Add keywords for one source or for all sources
2. Run `Last 24h` or search a custom window
3. Open paper links, send PDFs, and bookmark useful results
4. Turn recaps on and choose recap times in the user's local timezone

Slash commands are also available, including `/start`, `/help`, `/today`, `/keywords`, `/searchhours`, `/bookmarks`, `/dailyrecap`, `/setrecaptime`, `/report`, and `/refresh`.

## Quick start

This repository currently ships as a single-file Python bot.

```bash
conda activate main
pip install -r requirements.txt
export TELEGRAM_BOT_TOKEN="123456789:AA..."
python arXiv_bot.py
```

The bot runs with **Telegram long polling**, so no webhook setup is required.

## Configuration

### Required

- `TELEGRAM_BOT_TOKEN`: token generated by BotFather

### Recommended

- `OPENALEX_MAILTO`: contact email sent with OpenAlex-backed requests

### Optional

- `MAX_RESULTS`: cap on fetched results before filtering
- `RESULTS_PAGE_SIZE`: how many papers to show per Telegram page
- `MAX_ABSTRACT_CHARS`: abstract preview length
- `REPORT_FORWARD_CHAT_ID`: enables the in-bot issue reporting flow
- `REPORT_ADMIN_USER_ID`: limits admin-only report target commands
- `COFFEE_URL`, `COFFEE_TEXT`, `COFFEE_EVM_ADDRESS`, `COFFEE_SOLANA_ADDRESS`, `COFFEE_BTC_ADDRESS`: support links and donation addresses

You can also preseed default keyword lists through environment variables such as:

- `ARXIV_KEYWORDS`
- `BIORXIV_KEYWORDS`
- `MEDRXIV_KEYWORDS`
- `CHEMRXIV_KEYWORDS`
- `SSRN_KEYWORDS`
- `IEEE_KEYWORDS`
- `PUBMED_KEYWORDS`

These values should be comma-separated.

## Project layout

- `arXiv_bot.py`: main Telegram bot, fetchers, scheduling, and formatting logic
- `bot_settings.json`: local runtime state for users, keywords, bookmarks, and recap settings
- `requirements.txt`: Python dependencies
- `AGENTS.md`: contributor notes for working on the repository

`bot_settings.json` is updated at runtime and should be treated as local state, not as a source-of-truth configuration file.

## Notes for deployment

- The bot stores per-user state in a local JSON file, so persistent storage matters if you deploy it on a server.
- Do not commit real bot tokens, personal chat IDs, or production user data.
- A quick sanity check before deployment is:

```bash
python -m py_compile arXiv_bot.py
```
